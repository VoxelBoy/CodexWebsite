Faster Prototyping with Unity's DOTS Stack

Unity’s Data-Oriented Technology Stack has matured to the point where I use it for the earliest prototypes, not just end-game optimizations. The ability to express gameplay as data transformations unlocks workflows that are impossible with traditional, component-driven hierarchies.

Compose systems in layers

I start every prototype with a thin IAspect layer that exposes the data a designer actually cares about. Gameplay systems then operate on the aspect, not raw components. Here’s a simplified example from our vehicle swarm tests:

public readonly partial struct VehicleAspect : IAspect
{
    public readonly RefRO<VehicleStats> Stats;
    public readonly RefRW<SteeringState> Steering;
    public readonly RefRW<LocalTransform> Transform;

    public void Integrate(float deltaTime)
    {
        var steering = Steering.ValueRW;
        steering.Velocity += steering.Acceleration * deltaTime;
        steering.Velocity = math.clamp(steering.Velocity, 0f, Stats.ValueRO.MaxSpeed);
        Transform.ValueRW.Position += steering.Velocity * steering.Direction * deltaTime;
    }
}

Designers tweak VehicleStats in a structured authoring window, and automated tests assert bounds so we never publish unsafe values.

Jobs drive the iteration loop

The Burst compiler gives immediate feedback: if a new behaviour introduces an allocation or sync point, the job debugger screams at you. We integrate this into CI so every pull request validates job safety. When we want to visualize data, an editor-only SystemAPI.Query draws gizmos that never ship to players.

Hybrid components where they matter

Not every subsystem is ready for DOTS. Audio, UI, and bespoke physics stay on GameObjects until the performance profile says otherwise. Hybrid components give me the runway to migrate pieces gradually without breaking designers’ prefabs.

If you’re modernizing a long-running Unity project, adopting DOTS incrementally keeps momentum high. Start with the systems that gate your iteration speed, enforce Burst-friendly coding standards, and layer in automated validation early.
